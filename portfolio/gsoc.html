Parallelization In MOOSE


Google Summer of Code Project


Neuronal and multiscale simulations are computationally demanding, they require large number of very similar calculations. One of the core problems in this domain is to rapidly perform detailed single-neuron calculations. In MOOSE (Multiscale Object-Oriented Simulation Environment) one such solver is “KSOLVE”. The two methods I worked on for parallelization were OpenMP and NVIDIA Cuda®. Results obtained from OpenMP showed very less to no improvement in computation time. It was then parallelized using Cuda® and a signiﬁcant reduction in computation time was observed.


The image above shows the results obtained by using OpenMP. It actually gave a slow down as the time taken for the thread creation and destruction dominated over the time saved due to parallel computation. Several strategies were applied but no success. Then I shifted to using NVIDIA Cuda®.

The image abive shows the results of using Cuda. A significant amount of speed up was obtained. Separate "rk4" solver for the device(GPU) was written in Cuda C++. The full details can be found here.
